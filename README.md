## 📌 Overview

The website features interactive technical articles where static equations come to life.

## 🎯 Purpose

There were various reasons why I built this website, but the mainly ones were:

- **Democratizing access to high-quality content in Brazil:** Providing it lacks so much technical content with intuition.
- **Decrease complexity barrier:** Helping enthusiasts and students master complex topics in a way that feels intuitive and precise.
- **Cultivate an Open Academic Culture:** Encouraging people to share academic and technical knowledge on the Internet, fostering a community that builds and teaches.

## ⚙️ Pipeline

In this section, I explain the architecture of Kaga and my design decisions.

### </> Website

The blog was built using [Vitepress](https://vitepress.dev/) for three main reasons:

- SSG (Static Site Generation);
- The articles can be written in Markdown and they automatically become web pages;
- It allows the direct use of [Vue.js](https://vuejs.org/) logic inside Markdown.

The styling and theming were implemented using [Tailwind](https://tailwindcss.com/). Following a mobile-first approach.

The Presentation were generated by [Manim Slides](https://www.manim.community/plugin/manim-slides/), built on top of [Manim](https://www.manim.community/). It generates various .mp4 files, each of which becomes a slide in the presentation, and combine them into a single .html file that could be embedded in an iframe.

All interactivity — the Knowledge Gates and the Presentation — were implemented using plain Vue.js.

The website is designed to run smoothly even in slow connections. Hence, only the essenail javascript is loaded. For example, the Youtube Video Player is initially displayed as a static image. The actual iframe is only created when the user clicks the play button, reducing unecessary requests when the user doesn't want to watch the video.

I deployed the website using [Vercel](https://vercel.com/) and it is available [here](https://kaga-blog.vercel.app/). 

### ▶️ Youtube

I use [Manim](https://www.manim.community/) — a very powerful Python library that creates high-quality animations — to generate the animations in my videos.

The voice is mine (sorry for any accent or misspronunciation).

The videos are posted in my [youtube channel](https://www.youtube.com/@adriytkr).

### 📝 Scripts

For each video, I would typically follow this workflow:

1. I begin choosing a topic, preferably one that is visual, but even abstract topics can be visualized.

2. I write the script for the youtube video, my lines and what my animations should look like. It is very abstract at this moment.

3. I code the animations using Manim and then export the scenes as .mp4 files — either as a single file, or multiple files, depending on the video.

4. I record my lines using my phone's built-in voice recorder. Then, I import the audios into [Audacity](https://www.audacityteam.org/) to process them — removing background noise, compressing.

5. With the animations and audios done, I import them into [Kdenlive](https://kdenlive.org/). After the editing is done, I export the video as .mp4 file and prepare to upload the video on Youtube.

6. The work is mostly done. I use [paint.net](https://www.getpaint.net/) to create the thumbnail, add subtitles, write the description and finally hit upload.

7. However, I am not done yet. I still need to write my article. I usually follow a template
    1. Pre-requisites
    2. Intuition (Visual Proofs)
    3. Formal Proofs

8. Sometimes, an article requires a Presentation, so I use Manim Slides to code the scenes and export them as an iframe.
